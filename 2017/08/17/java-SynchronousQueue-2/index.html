SynchronousQueue精读(2)-queue

首先SynchronousQueue源码基于jdk-1.7
SynchronousQueue是无锁实现的阻塞队列。
queue实现的是公平模式，包含一个head和tail指针，初始时head = tail，head.next = null：

  1. 插入在tail插入。
  2. 取出在head取出。

所以实现fifo，先进先出的顺序。

Qnode

  1. next：指向队列中的下一个结点。当将结点从队列中移除，则将next指向自身。
  2. item：存放的数据，如果item指向自身结点引用，则表示取消结点操作。
  3. waiter：操作指向的线程。持有这个对象，最要为了阻塞和唤醒。
  4. isData：表示是获取(poll()和take())操作，还是offer()和put()的插入操作。

首先除了isData之外，都是volatile类型，跟stack一样是为了可见性。
isData是final类型，所以是不可变对象，线程安全。

Qnode和Snode区别

区别：

  1. Snode多了一个mode域，表示REQUEST、DATA和FULLFILLING操作。
  2. Qnode没有mode，而是用一个isData域来替代，因为Queue没有FULLFILLING这一阶段。Stack有匹配这个阶段主要是因为，栈的插入和获取都在head进行，所以并发情况比较多，不能靠一次CAS就可以完成，所以多了一个FULLFILLING阶段。而Queue里面插入和获取不再同一个结点，所以可以减少这个阶段。
  3. Snode中结点取消，用metch域指向自身结点标识。
  4. 而在Qnode中，结点操作取消用item指向自身标识。
  5. Qnode没有tryMatch()操作。当需要与head处的结点进行配对时，取出目标结点的item域，然后移动head指向目标结点(移除)。

构造方法

设置item和isData。

casNext()

CAS更新结点的next域。

casItem()

CAS更新结点的item域。

tryCancel()

将结点的item域指向自身，表示结点操作取消。

isCancelled()

如果结点的item域指向自身结点，则表示结点操作取消。

isOffList()

如果结点的next域指向自身，则表示结点不在队列上。

Queue成员变量和方法

成员变量

  1. head：队列头结点，头结点是一个哑结点。
  2. tail：队列尾结点。
  3. cleanMe：记录上一个取消结点的前驱。在clean()方法中被使用。删除上一个取消的结点。

这三个域都是volatile的，是为了可见性。

构造方法

创建一个队列头哑结点。head = tail，head.next = null。

advanceHead()

CAS更新head指向nh。另外让原来的head的next域指向自身。

advanceTail()

CAS更新tail域为nt。

casCleanMe()

CAS更新cleanMe域。

transfer()

首先判断操作类型，然后主要分为两部分：

  1. 如果队列为空，或者操作与队列tail尾部的操作相同，则将操作，线程对象封装成Qnode用CAS插入到队列尾部，插入成功后，则调用awaitFulfill()自旋或者阻塞等待匹配。
  2. 否则，进行匹配，取出head.next的结点，记为m，获取m结点的item，并且将m的item设置为e(本次操作的item域)，最后更新head指向目标结点m进行移除。

注意transfer()方法的返回值：

  1. 对于放入数据的操作，如果成功则返回自己的obejct。超时返回null。
  2. 对于获取数据的操作，如果成功则返回对方的Object。超时返回null。

awaitFulfill()

与stack中的自旋阻塞等待方法一样。

clean()

  1. 如果head.next已经取消，则清除。
  2. tail发生并发修改，则更新。
  3. 如果本次需要删除的参数s结点不是tail，则删除返回。
  4. 如果cleanMe为null，则将pred保存到cleanMe。
  5. 否则删除上一次记录的cleanMe.next。
