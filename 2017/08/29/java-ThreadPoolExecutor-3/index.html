ThreadPoolExecutor精读(3)-提交任务

首先ThreadPoolExecutor源码基于jdk-1.7

内容

  1. addWorker()：创建新的工作线程。
  2. getTask()：从任务队列中获取任务。
  3. processWorkerExit()：工作线程退出的处理。
  4. runWorker()：工作线程不断从任务队列获取任务，然后执行。
  5. execute()：将任务提交到线程池。

addWorker()

两个参数：

  1. firstTask：表示新创建的工作线程，第一个需要执行的任务。
  2. core：表示新创建的工作是否属于核心线程。

两个返回值：

  1. true：表示新的工作线程创建成功。
  2. false：表示工作线程创建失败。

addWorker()的三个过程

首先将前面两个检查过程放入一个for死循环中，处理CAS更新失败的情况，然后分为如下三个步骤：

  1. 线程池的状态检查是否允许创建新的工作线程。
  2. 通过检查线程池的工作线程数量，与corePoolSize和MaxPoolSize的关系决定是否创建。
  3. 加锁更新统计变量，创建新的工作线程，并且启动工作线程。

线程池状态

在获取线程池状态rs之后，分为5种情况：

  1. 线程池状态为RUNNING，则表示可以允许创建新线程，则进入下一步判断。
  2. 线程池状态为STOP、TIDYING或者TERMINATE，不能再执行任务，所以没有必要再创建新的工作线程。则返回false。
  3. 线程池状态为SHUTDOWN，但是firstTask != null，则表明需要新创建的工作线程不是替代线程，所以是一个提交任务导致的线程创建，由于线程池状态为SHUTDOWN，不再接收任务提交，所以返回false。
  4. 线程池状态为SHUTDOWN，且firstTask == null，但是任务队列为空。这里firstTask == null表示创建的是一个替代线程(或者扩容线程)，但是任务队列已经为空，所以也没有理由创建新的工作线程，返回false。
  5. 线程池状态为SHUTDOWN，且firstTask == null，且任务队列不为空。SHUTDOWN状态虽然不再接收新任务，但是会将任务队列中的任务全部执行完再关闭。由于工作线程会由于任务执行异常而退出，所以创建新的工作线程是被允许的。则进入下一步判断。

检查线程池工作线程的数量

首先获取线程池当前的工作线程池数量wc，然后根据core值，分三种情况判断：

  1. 如果wc超过CAPACITY，即线程池最大工作线程数量(2^29 - 1)，则不允许创建，直接返回false。
  2. 否则，如果core == true，表示创建的是核心池的线程，所以wc只要小于corePoolSize就允许创建。否则如果wc > corePoolSize，则不允许创建，返回false。
  3. 否则，如果core == false，表示创建的不是核心池的线程，所以wc只要小于maxPoolSize就允许创建。否则，如果wc > maxPoolSize，则不允许创建，返回false。

在判断结束后，如果没有返回false。则允许创建新的工作线程，分为两种情况：

  1. 如果CAS更新wc成功，则继续下面。
  2. 否则CAS更新wc失败，则返回到最开始的retry，重新判断。

创建并且启动新的工作线程

  1. 使用threadFactory创建新的工作线程。然后获取mainLock。
  2. 如果创建工作线程失败，或者线程池状态检查(过程1)不允许创建新线程，则恢复wc，调用tryTerminate()方法，返回false。
  3. 否则工作线程创建成功，而且线程池状态允许使用新线程，则将新的工作线程加入到works，即工作集合中。
  4. 更新统计变量。
  5. 启动线程。
  6. 重置中断中断，并且只有在线程池状态为STOP的情况下才允许工作线程的中断置位。目的是为了让工作线程在线程池状态为RUNNING和SHUTDOWN时，使线程执行任务不被中断影响，正常执行。
  7. 返回true，表示创建成功。

getTask()

getTask()：一个工作线程从任务队列取出任务，直到工作线程异常退出，或者超时退出。
getTask()将三个过程放在一个for死循环中，然后getTask()执行包含三个过程：

  1. 线程池状态检查。
  2. 线程空闲超时检查，以判断是否退出线程。
  3. 从任务队列中获取任务。

线程池状态检查

线程池检查分为3种情况：

  1. 如果线程池状态为STOP、TIDYING或者TERMINATE，则更新线程池的wc，线程终止，即返回null。
  2. 如果线程池状态为SHUTDOWN，而且任务队列为空，则由于SHUTDOWN不再接收新的任务，所以线程没有任务可以执行，线程终止，即返回null。
  3. 否则说明需要继续从任务队列取任务。

线程空闲超时退出检查

  1. 获取线程池当前的wc。
  2. 确定timed，即是否采用空闲超时退出机制。当allowCoreThreadTimeOut = true或者线程池当前工作线程数大于corePoolSize时，time = true，即开启这种机制。
  3. 接下来判断线程是否需要终止：
    3.1 如果当前wc > maxPoolSize，则终止当前线程。
    3.2 否则wc <= maxPoolSize，且如果上一次获取任务没有超时，则break，进行第三步。
    3.3 否则wc <= maxPoolSize，且如果上一次获取任务超时了，但是timed = false，则进行第三步。
    3.4 否则wc <= maxPoolSize，且如果上一次获取任务超时了，而且timed = true，则终止当前线程。

从任务队列中获取任务

  1. 根据timed，即是否考虑空闲超时退出机制，确定从任务队列获取任务的方法。如果timed = true，则使用timeout版本的poll()方法，获取任务。否则使用take()即一直阻塞的方法获取任务。
  2. 如果容队列获取返回，获取的任务不为null，则返回因为获取了任务。
  3. 否则没有获取任务，则置timedOut = true，等待下次回到第二个过程终止线程。

processWorkerExit()

processWorkerExit()：在runWorker()中当线程在执行任务中遭遇异常，或者任务队列为空时调用。即一个工作线程退出时被调用。分为两种退出：

  1. 任务队列为空，或者空闲超时，则属于线程正常退出。
  2. 否则在工作线程执行任务中，发生异常，则属于异常退出。异常退出需要创建替代线程。

processWorkerExit()有2个参数：

  1. Worker w：表示工作线程。
  2. completedAbruptly：false表示这是正常退出，true表示异常退出，即线程执行任务时发生异常。如果是异常退出，则在processWorkerExit()最后需要创建替代线程，代替这个异常退出的线程。

processWorkerExit()执行包括三个过程：

  1. 更新统计变量和线程池wc。
  2. 调用tryTerminate()尝试将线程池状态置为TERMINATE。因为每次线程终止都有可能使线程池关闭。
  3. 判断是否需要创建替代线程。

更新统计变量和线程池wc

  1. 如果completedAbruptly = true，则工作线程是正常退出，所以调用decrementWorkerCount()减小wc。 否则completedAbruptly = false，则工作线程是异常退出，在后面需要创建替代线程，所以不需要减小。
  2. 获取maiLock更新统计变量completedTaskCount，并且将当前工作线程从worker集合中移除。

调用tryTerminate()

tryTerminate判断如果线程池当前情况为下面两种时：

  1. 线程池状态为SHUTDOWN，而且线程池活动线程为0，且队列为空。
  2. 线程池状态为STOP，而且线程池活动线程为0。

这两种情况需要将线程池状态置为TIDYING，然后调用hooks扩展方法terminate()。
最后从terminate()方法返回后，将线程池状态置为TERMINATE。

如果线程池状态不为上面两种，也分为两种情况：

  1. 任务队列为不为空，则直接返回。
  2. 任务队列为空，则中断线程池中一个空闲的线程。

判断是否需要创建替代线程

  1. 如果线程池状态为STOP、TIDYING或者TERMINATE，则没有任务需要执行，所以不需要创建替代线程，返回。
  2. 否则线程池状态为RUNNING或者SHUTDOWN，且如果是异常退出，则明确需要创建替代线程。
  3. 否则线程池状态为RUNNING或者SHUTDOWN，且如果是正常退出，再判断allowCoreThreadTimeOut。这里分为三种情况：
    2.1 如果allowCoreThreadTimeOut = true，且任务队列为空，则无论怎么样都不会创建替代线程。
    2.2 如果allowCoreThreadTimeOut = true，但是任务队列不为空，则如果此时线程池wc大于等于1，则不需要创建替代线程。否则创建替代线程。即此时最大保留一个线程。
    2.3 如果allowCoreThreadTimeOut = false，且当前线程数大于等于corePoolSize，则不需要创建。否则当前线程数小于corePoolSize则创建替代线程。
  4. 创建替代线程。

这里替代线程的创建策略总结为(在线程池状态为RUNNING或者SHUTDOWN条件下)：

  1. 如果是异常退出，则肯定创建替代线程。
  2. 如果是正常退出，且allowCoreThreadTimeOut = true，即核心线程也会空闲超时退出，当任务队列还有任务时，则只有当线程数为0时，才创建替代线程，防止没有线程执行任务。
  3. 如果是正常退出，且allowCoreThreadTimeOut = false时，只有当当前线程数小于corePoolSize时，才创建替代线程。

runWorker()

runWorker()：不断从队列中取任务，直到队列为空，或者任务执行异常退出。

  1. 在while()循环中取任务，如果取出null则说明对立为空，将completedAbruptly置为false，调用processWorkerExit(w, completedAbruptly);统计清理，终止线程。
  2. 如果取出来的任务不为null，则首先调用beforeExecute()，然后执行认为有，最后任务完成再调用afterExecute()，更新统计量，重新取任务。
  3. 如果任务执行过程中发生异常，则更新统计量，最后调用processWorkerExit(w, completedAbruptly);进行异常退出清理，这里的completedAbruptly为true。

execute()

execute()执行分为三个过程：

  1. 如果线程池中线程的数量小于corePoolSize，则创建一个新的线程执行任。
  2. 否则插入到任务队列中，稍候执行。
  3. 如果任务队列已经满了，则新建一个线程执行。如果创建失败或者超过maxPoolSize，则使用拒绝策略拒绝任务。
