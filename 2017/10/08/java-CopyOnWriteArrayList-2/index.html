写操作

写操作包括：

  1. set：将list中指定位置的对象替换为element。
  2. add：在list尾部或者指定位置插入element。
  3. remove：删除list中指定位置的对象或者删除指定对象。
  4. addIfAbsent：如果list中不存在element，则插入到list尾部。

set

可以看出这个set写操作需要加锁。然后如下：

  1. 首先volatile读操作，获取最新的共享变量以及array数组对象。(与其它的写操作建立happen-before关系)。
  2. 然后分为两种情况，如果index位置的对象与element不相同，则进行复制，最后setArray()进行volatile写。(保证下次的读写操作可以看到所有共享变量的更新)。
  3. 否则index位置的对象与element相同，进行一次看似多余的setArray()操作，是一个volatile写操作，为了保证内存一致性。即之前提到的set之前的操作对后面读写操作可见，即不是list变量的可见性。
  4. 最后释放锁，返回旧值。

add

add方法有两种：

  1. add(e)：将对象插入到list尾部。
  2. add(index, e)：将对象插入到index位置，原来index位置及之后的元素向后移动移位。

add(e)

在list末尾添加e对象，同时保持两个可见性：

  1. list对象数组的可见性。
  2. 其它共享变量的可见性。

add(index, element)

将指定的对象插入到指定的位置，list中指定位置后面的对象向后移动一个位置(并没有真正移位 而是通过复制实现)。
可见性与add(e)方法一样。

remove

remove方法有两种：

  1. remove(index)：移除index位置的对象，index后面的对象向前移动一位。
  2. remove(o)：在list中移除o，如果不存在返回false，存在返回true。
  3. 获取lock锁。
  4. volatile读，获取array即对象数组。
  5. 分为两种情况，如果index是数组最后一个位置时，即不需要移动，则直接复制。
  6. 否则需要移动，则创建新的对象数组，分别复制前半部分和后半部分。
  7. 然后setArray进行volatile写。
  8. 最后释放独占锁，返回旧值。

remove(index)方法始终会执行setArray()方法，即volatile写操作，主要为了两个实现其它共享变量的可见性，即内存一致性。与add方法一致。

remove(o)

与remove(index)类似，但是不一定能保证内存一致性，因为如果o不存在则不会执行setArray()的volatile写的操作。但是在o存在的情况下，可以保证内存一致性。

addIfAbsent

addIfAbsent(e)：如果e当前不在list中，则将e插入到list尾部。插入成功返回true，失败j即已经存在则返回false。

整体思路是边遍历边复制，如果遍历遇到目标e，则返回false。
与remove(o)类似，当不需要对list进行修改时，不会保证内存一致性。

集合类操作

集合类操作包括五种：

  1. containsAll(Collection c)：判断c中的对象是否都在list中。
  2. removeAll(Collection c)：在list中删除所有在c中的对象。
  3. addAll(Collection c)：在list的尾部逐个插入c中的对象。
  4. retainAll(Collection c)：在list中移除除了c中对象以外的对象。
  5. addAllAbsent(Collection c)：将c中本来不在list的对象都插入到list尾部。

上面5种方法，除了第一种是读操作以外，其它都是写。
这4种写操作与remove(o)和addIfAbsent(e)一样，都不能保证内存一致性。
原因在于当这4中方法完成时，如果list没有发生变化，则不会执行setArray()的volatile写操作，此时没有内存一致性。

总结

CopyOnWriteArrayList具有以下特性：

  1. 读不加锁，写加锁。
  2. 通过写时复制实现，读写并发。
  3. 在add、set和remove(index)方法可以保证多线程内存一致性。
  4. 遍历不会抛出ConcurrentModificationException异常。
  5. 遍历使用listIterator迭代器，支持向前与向后的遍历。
  6. 读写并发操作的可见性由array的volatile读和volatile写建立happen-before关系实现。
  7. 内存一致性也由array的volatile读和volatile写保证所有共享变量的可见性。
